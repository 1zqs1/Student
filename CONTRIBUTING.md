贡献指南（CONTRIBUTING）

目标：本项目用于共同创建个人简介网页端，所有贡献者请遵循以下流程以便协作与审查。

工作流程：
1. Fork 本仓库到你的账户。  
2. 从你的 Fork 创建新的分支（命名建议：`feature/你的姓名-简述`）。  
3. 在你的分支中复制 `template/` 到 `测试X/`（或在已有 `测试X/` 中创建子目录，如 `studentA/`），进行开发和调试。  
4. 提交清晰的 commit，推送到你的 Fork。  
5. 在原仓库创建 Pull Request（目标分支为 `main` 或指定的协作分支）。  
6. 在 PR 中描述你的更改、目的与演示方式。所有参与者应在 PR 中讨论并达成共识。  
7. 通过审查并达成一致后，由维护者或指定负责人合并 PR。  
8. 合并后，维护者可将最终版本复制到 `测试结果/` 目录作为交付物，并在 `测试结果/README.md` 中记录作者与版本说明。

注意事项：
- 请勿直接在 `main` 分支上修改 `测试1/`、`测试2/`、`测试3/` 的源文件。  
- 若需要紧急修复，请在 PR 中标注并联系维护者。  
- 保持提交信息简洁有意义，PR 描述应包含测试步骤与预期效果。

如需变更协作流程或指定负责人，请在 Issue 中提出讨论。
