# Student

该项目用于测试与学习。项目目的是为学习者提供一个简单的练习与验证环境，方便进行功能测试、代码练习和学习记录。

主要用途：
- 测试小型代码示例和练习。
- 记录学习过程与笔记。
- 作为教学或实验的最小示例仓库。

文件说明：
- HTML 文件（`.html`）：用于构建页面结构与展示内容，适合练习静态页面布局、语义标签与基础页面结构。
- JavaScript 文件（`.js`）：用于实现页面交互与业务逻辑，适合练习 DOM 操作、事件处理、表单验证和简单脚本逻辑。

示例用途：
- 将 HTML 与 JS 配合用于演示页面交互（按钮点击、表单提交、动态内容更新）。
- 作为前端入门练习的最小可运行示例，便于学习和测试浏览器端功能。

模板与使用流程：
- 使用 `template/` 目录下的文件作为起点，复制到各自的 `测试X/` 目录进行实现与修改。
- 各同学在 `测试1/`、`测试2/`、`测试3/` 中基于模板开发，完成后将最终通过测试的代码放到 `测试结果/`。
- 为防止直接更改主分支内容，请通过 Fork + PR 的方式提交修改（或在 GitHub 上为 `main` 开启分支保护）。

实验目的：
我们需要共同创建一个个人简介网页端，展示个人信息、技能和联系方式，最终效果以协商为主。具体目标：
- 提供一个简洁的个人简介页面模板，包含头像、简介、技能列表、项目链接与联系方式。
- 支持在浏览器端展示交互（例如切换主题、展开详细信息或表单提交）。
- 各成员基于 `template/` 开发自己的版本，最终通过讨论与评审达成一致后，将最终代码放入 `测试结果/` 作为交付版本。

协作原则：
- 代码修改通过 Fork + Pull Request 流程进行，所有重要更改需通过讨论和审查以达成共识后合并。
- 在 `测试X/` 中进行个人开发，保留原始模板不被直接覆盖；合并到主分支前请在 PR 描述中说明修改内容与作者。
- 最终效果由参与者协商决定，争议通过评论区讨论或指定项目负责人裁定。

文件夹约定：
- `测试1/`、`测试2/`、`测试3/`：分别放置三位同学或三份不同的测试代码，每个文件夹内按个人或版本组织代码文件（例如 `index.html`、`app.js`、资源文件等）。
- `测试结果/`：用于放置最终汇总或通过测试后的最终代码版本，作为该次练习或实验的交付物。

建议：
- 在各 `测试` 文件夹内使用清晰的命名（例如 `studentA/`、`studentB/` 或 `v1/`、`v2/`）以便区分作者与版本。
- 将最终通过的代码复制或移动到 `测试结果/`，并在 `README.md` 中记录每次提交的说明与作者。
